High

Windows detect() can't open a directory handle✓ verified

crates/decmpfs/src/windows.rs:73

open_with calls CreateFileW with dwFlagsAndAttributes = 0, which can't open a directory. A fresh install probes the parent dir → ERROR_ACCESS_DENIED → the Err(_) folds into the Unsupported arm, so every fresh NTFS install writes plain and misreports the reason.

Fix: pass FILE_FLAG_BACKUP_SEMANTICS, or detect via volume-root APIs.

classify_skip maps only POSIX errno for Busy✓ verified

crates/decmpfs/src/safety.rs:105

Not cfg-gated, so on Windows ERROR_SHARING_VIOLATION (32) → None → apply_guarded propagates a hard Err, breaking the fail-soft contract. remove.rs::retryable already encodes 32/33.

Fix: a #[cfg(windows)] errno arm mapping 32/33 → Busy. Landed (c0f008e).

build_resource_fork emits a malformed table for zero-length content✓ verified

crates/decmpfs/src/macos.rs:152

Empty input → num_blocks = 1 but chunks yields zero blocks, so the loop writes [8,0,0,0] while offset[0]=8 points past a 4-byte blob, then UF_COMPRESSED + rename over the original.

Fix: special-case raw.is_empty() → well-formed single-entry table + invariant test. Landed (c0f008e).

napi/decmpfs/src/lib.rs:583

napi/decmpfs/src/lib.rs:133

restore() swallows rollback failures

crates/decmpfs/src/safety.rs:171

An ENOSPC mid-rollback still returned Ok(Skipped), leaving a corrupt file reported benign. Fix landed (c0f008e): returns Result, callers propagate.

What's solid

The fail-soft contract is real: the decmpfs path caps its u32 offset math with within_decmpfs_limit and documents the overflow class; remove.rs::retryable already encodes the Windows sharing/lock codes; classify_skip is ErrorKind-first; rollback plus an unconditional plain-write fallback net most corruption/skip paths; publish-crate.mts is a clean idempotent-publish reference. The theme is narrow — Windows/POSIX errno divergence and a couple of untested-by-default edges — not a structural weakness in the codec design.
